# Claude Code 設定

## プランモード終了後のタスクリスト

プランモードを抜けたら、以下のタスクを順番に実行すること：

1. **プランファイルの命名を変更する**
   - 自動生成された名前（例: `streamed-riding-finch.md`）を、日付ベースの分かりやすい名前に変更する
   - 命名規則: `YYYY-MM-DD-<内容の簡潔な説明>.md`
   - 例: `2026-02-03-readme-fix.md`, `2026-02-03-add-feature.md`

2. プランの実装
   - プランに従って実装を進める

## エージェントチーム編成

並行して実行可能なタスクリストの場合、エージェントチーム編成を積極的に提案すること。
- plan モード有効無効は問わない
- 各エージェントに明確な役割を割り当てる
- 利用するエージェントは汎用的なものでも構わない
- リーダー、メンバーは軍隊上の上官、下士官の関係を想定する。チーム内コミュニケーションは上下関係を考慮した上で軍隊用語を使用すること

### チーム解散時の必須手順（厳守）

**TeamDelete を実行する前に、必ず以下を実行すること。これはスキップ不可の必須手順である。**

1. **ユーザーのレビュー・承認を待つ**: チームの成果物に対してユーザーが確認・レビューを完了するまで、絶対にチームを解散しない。ユーザーから明示的に解散の許可が出るまで隊員は待機させておくこと
2. **行動記録の保存**: 各隊員の行動サマリー（担当タスク、実行内容、成果、発見事項）を専用ログファイルに書き出す
   - 保存先: `plans/YYYY-MM-DD-<チーム名>-log.md` として独立したファイルを作成する（プランファイルには追記しない）
   - 含めるべき情報: 隊員名、担当タスク、主要な調査結果、修正内容、発生した問題と対処
3. **TeamDelete の実行**: 上記1・2が完了してから TeamDelete を実行する

### 隊員プロンプトの必須事項

隊員を spawn する際、プロンプトに以下を必ず含めること:
- 「完了報告は1回のみ。反復報告は不要。送信後は沈黙を保て」
- ファイル編集を伴うタスクでは、フェーズ分離によるファイル競合防止策を明記
